## 1. Major Strengths

- **Committed data and results.** All four canonical routing datasets
  (`data/real_*_routing_dataset_enriched.csv`) and all main-paper output
  directories are committed, enabling full offline reproduction without API
  calls.
- **Clear canonical policy story.** `adaptive_policy_v5` (primary),
  `v6`, and `v7` are clearly designated as the manuscript-facing policies in
  `docs/CANONICAL_MANUSCRIPT_DECISIONS.md`.
- **Offline reproducibility.** Offline analysis paths are well-documented in
  `REPRODUCIBILITY.md` and `MANUSCRIPT_REPRODUCTION.md`; the test suite (677
  tests, 0 failures) validates the pipeline without any API key.
- **Separation of exploratory content.** Most exploratory/blocked artifacts are
  already marked as `🔬 Exploratory` or `❌ Blocked`, preventing accidental
  citation.
- **`docs/internal/` already present.** AI-agent planning logs were already
  moved to `docs/internal/` before this pass, which is good practice.
- **Final canonical artifact directories exist.** `outputs/paper_tables_final/`
  and `outputs/paper_figures_final/` contain the definitive manuscript assets
  regenerated by `scripts/generate_final_manuscript_artifacts.py`.
- **Security.** No hardcoded API keys found; `.env` is gitignored; `.env.example`
  provided.

## 2. Major Weaknesses (Pre-Cleanup)

### W1. Root-level document clutter (HIGH)
Three root-level files were marked "OUTDATED / SUPERSEDED (2026-03-31)" in their
own banners but remained at the top level, confusing the entry-point hierarchy:
- `MANUSCRIPT_ARTIFACTS.md`
- `MANUSCRIPT_SUPPORT_ADDITIONS.md`
- `PAPER_ARTIFACT_STATUS.md`

**Resolution (this pass):** Moved to `docs/archive/`.

### W2. `paper_tables_cleaned` vs `paper_tables_final` inconsistency (HIGH)
`MANUSCRIPT_REPRODUCTION.md` Section 3 explicitly listed `paper_tables_cleaned/`
and `paper_figures_cleaned/` as the canonical citation targets, contradicting the
banner at the top of the same file and contradicting `FINAL_MANUSCRIPT_QUICKSTART.md`,
`FINAL_MANUSCRIPT_ASSET_INDEX.md`, and `docs/CANONICAL_MANUSCRIPT_DECISIONS.md`,
which all point to `paper_tables_final/` and `paper_figures_final/`.

The README `Committed Results` table also listed `paper_tables_cleaned/` as
`✅ Main paper`.

**Resolution (this pass):**
- `MANUSCRIPT_REPRODUCTION.md` Section 3 rewritten to use `paper_tables_final/`
  and `paper_figures_final/` as canonical, with `paper_tables_cleaned/` demoted
  to a "historical intermediates" sub-table.
- `README.md` Committed Results table updated: `paper_tables_final/` and
  `paper_figures_final/` are now listed as `✅ Main paper`; `paper_tables_cleaned/`
  and `paper_figures_cleaned/` marked `📦 Historical`.

### W3. Broken path in `REPRODUCIBILITY.md` (MEDIUM)
`REPRODUCIBILITY.md` step A6 referenced `outputs/real_budget_sweep/` as the
output path; the actual committed directory is `outputs/budget_sweep/`.

**Resolution (this pass):** Corrected to `outputs/budget_sweep/`.

### W4. Non-existent script referenced in `REPRODUCIBILITY.md` (MEDIUM)
`REPRODUCIBILITY.md` step B7 referenced `scripts/run_paper_table_export.py`
with a comment `# if available` — the script does not exist.

**Resolution (this pass):** Replaced with the correct scripts:
`generate_paper_tables.py`, `generate_paper_figures.py`, and
`generate_final_manuscript_artifacts.py`.

### W5. `PUBLIC_RELEASE_CHECKLIST.md` at root (LOW)
A process/housekeeping document appeared at the repository root alongside the
manuscript-facing README, adding noise for external readers.

**Resolution (this pass):** Moved to `docs/PUBLIC_RELEASE_CHECKLIST.md`.

### W6. `docs/` directory has ~90 files with no navigation index (MEDIUM)
The `docs/` directory contained ~90 markdown files ranging from canonical
manuscript documents to exploratory working notes and superseded analyses, with
no clear separation for external readers.

**Resolution (this pass):** Created `docs/README.md` with a four-tier index
(Canonical, Supporting Reference, Internal, Archive) to help readers navigate
without moving all the working notes.

### W7. `docs/archive/` directory did not exist (LOW)
The `docs/internal/` pattern was in place, but no `archive/` equivalent existed
for superseded non-internal documents.

**Resolution (this pass):** Created `docs/archive/` with a `README.md`.
